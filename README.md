# Data Fusion Contest 2026 - 3
решение для Задача 3 "Герои"

жадный алгоритм с поиском ближайшего соседа, учитывающий временные окна и стоимость найма.

Основная стратегия: для каждого героя мы ищем объект, который он успеет посетить вовремя (или подождать), минимизируя затраты очков хода и учитывая, что нанимать героев нужно последовательно.

#Почему это решение эффективно:
Соблюдение ограничений: Код учитывает visit_cost, перенос очков хода на следующий день и жесткие окна day_open.

Правило "Последнего шага": В логике проверки t_pts < v_cost заложено условие, что если очки есть, но их меньше 100, посещение засчитывается «на флажке».

Таверна: Алгоритм перебирает героев по порядку. Если герой не окупает свои 2500 золота (стоимость найма), он все равно может быть включен, если за ним стоят более выгодные герои, но поиск прекращается, если длинная череда героев становится убыточной.

Приоритет времени: Сортировка кандидатов (t_day * 10000) + d заставляет героев сначала собирать то, что исчезнет быстрее.
